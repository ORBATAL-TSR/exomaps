# LAN Server (07_LOCALRUN)

The entire `07_LOCALRUN/` directory is gitignored. Sets up production-grade HTTPS/HTTP2
serving for LAN clients (fixes Windows Chrome loading failures).

---

## Files

```
07_LOCALRUN/
├── setup.sh          One-time: install mkcert, Caddy, Gunicorn; generate TLS cert
├── run.sh            Start Gunicorn (Flask) + Caddy
├── rundaily.sh       Install as systemd service + cron watchdog
├── Caddyfile         Caddy config: HTTPS, HTTP2, /api proxy, SPA fallback
├── gunicorn.conf.py  Worker count, threads, timeouts
├── client-trust.sh   Install mkcert rootCA on client machines
├── dnsmasq.conf      Optional: resolve exomaps.local on LAN
└── certs/            Generated by setup.sh (gitignored)
    ├── exomaps.crt
    └── exomaps.key
```

---

## One-Time Setup (Server)

```bash
bash 07_LOCALRUN/setup.sh
```

- Detects LAN IP automatically
- Installs mkcert + generates local CA
- Generates TLS cert for `LAN_IP + exomaps.local + localhost`
- Installs Caddy (apt or brew)
- Installs Gunicorn + gevent

## Install as System Service (Auto-start on Boot)

```bash
bash 07_LOCALRUN/rundaily.sh
```

Creates systemd unit `exomaps-lan.service`:
- Starts on boot (after network-online.target)
- Restarts on crash (RestartSec=5)
- Logs to journald (`journalctl -u exomaps-lan -f`)

Also installs cron watchdog: checks every 15 min, relaunches if crashed.

### Service Control
```bash
sudo systemctl status exomaps-lan
sudo systemctl restart exomaps-lan
sudo journalctl -u exomaps-lan -n 50 --no-pager
```

## Gunicorn Config (gunicorn.conf.py)

```python
bind     = "127.0.0.1:5000"
workers  = (cpu_count * 2) + 1
threads  = 2
worker_class = "gthread"
keepalive    = 30
timeout      = 60
```

---

## Caddyfile Summary

- `auto_https off` — uses mkcert certs directly
- HTTPS on port 443, HTTP on 80 (redirects to HTTPS)
- Long cache headers for hashed assets (`immutable, max-age=31536000`)
- `/api/*` → Gunicorn proxy with 30s timeout
- SPA fallback: 404 → `index.html`

## Client Trust Setup (Each LAN Client)

```bash
# Run on each client machine (macOS/Linux)
bash 07_LOCALRUN/client-trust.sh

# Windows: manually import rootCA.pem via certmgr.msc
# → Trusted Root Certification Authorities
```

Chrome on Linux also needs NSS store update (done automatically by `client-trust.sh` via `certutil`).


---

## Why This Exists

`vite preview` uses HTTP/1.1 — only 6 concurrent connections per origin in Chrome.
With 210+ texture files, LAN clients would stall waiting for connections.

Caddy with HTTPS gives HTTP/2 multiplexing → all assets load in parallel.
Gunicorn replaces single-threaded Flask dev server → parallel API requests.
