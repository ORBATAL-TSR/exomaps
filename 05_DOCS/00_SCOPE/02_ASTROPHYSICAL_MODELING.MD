# 02 — Astrophysical Modeling & Inference

## Overview

ExoMaps uses real observational data as hard anchors, then fills gaps with constrained probabilistic models that remain **transparent**, **reversible**, and **reproducible**. Every inferred entity is tagged with its generation method, confidence tier, random seed, and model version so that science and simulation can coexist without confusion.

---

## Entity Confidence Tiers

| Tier | Label | Source | UI Treatment | Simulation Weight |
|------|-------|--------|-------------|-------------------|
| T1 | **Observed-Confirmed** | High-confidence published detections (NASA EA confirmed) | Solid icon, full detail | 1.0 (full trust) |
| T2 | **Observed-Candidate** | Unconfirmed detections with source-level confidence | Dashed outline, confidence badge | 0.5–0.9 (weighted) |
| T3 | **Inferred** | Generated by rule/model from partial observations | Ghost icon, "inferred" tag | 0.2–0.7 (model-dependent) |
| T4 | **Simulated-Decorative** | Optional low-priority detail for gameplay ambience | Faint icon, hidden by default | 0.0–0.1 (flavor only) |

### Confidence Score Composition

Each entity's confidence is a composite of three independent factors:

```
confidence = w_source × source_trust + w_complete × data_completeness + w_model × model_certainty
```

| Factor | Weight | Description |
|--------|--------|-------------|
| `source_trust` | 0.4 | Trust level of originating catalog (NASA EA = 0.95, SIMBAD = 0.85, etc.) |
| `data_completeness` | 0.3 | Fraction of key parameters present (mass, radius, period, SMA, ecc) |
| `model_certainty` | 0.3 | Inference model's self-reported uncertainty (1.0 for observed entities) |

## Planet Inference Pipeline

### Input Requirements
For each star system with fewer planets than statistically expected:
- Stellar mass, luminosity class, and metallicity (or estimates).
- Known planet positions (confirmed + candidate).
- System age proxy (if available).

### Inference Steps

1. **Expected Planet Count Estimation**
   - Use empirical occurrence rates by stellar type (e.g., Kepler/TESS statistics).
   - Adjust for metallicity: higher [Fe/H] → more giant planets.
   - Adjust for multiplicity: known multi-planet systems → higher expected count.

2. **Orbital Lane Placement**
   - Compute available zones between known planets.
   - Apply Titius-Bode–inspired spacing heuristics (log-spaced semi-major axes).
   - Respect Hill sphere exclusion zones around existing planets.
   - Enforce minimum orbital separation: Δa > 3.5 × mutual Hill radius.

3. **Planet Property Assignment**
   - Draw mass/radius from empirical distributions conditioned on orbital distance:
     - Inner zone (< 0.5 AU): rocky/super-Earth bias.
     - Habitable zone: Earth-like to sub-Neptune range.
     - Outer zone (> 3 AU): gas/ice giant bias.
   - Assign eccentricity from β-distribution with median ~0.05 for compact systems.
   - Estimate equilibrium temperature from stellar luminosity and SMA.

4. **Habitable Zone Classification**
   - Conservative HZ: 0.95–1.67 × √(L/L☉) AU (Kopparapu et al. 2013 limits).
   - Optimistic HZ: 0.75–1.77 × √(L/L☉) AU (early Mars / recent Venus limits).
   - Flag planets within HZ bounds with habitability likelihood score.

### Stability Validation
- Reject configurations with crossing orbits.
- Check for mean-motion resonance conflicts (2:1, 3:2, 5:3 ratios).
- Apply angular momentum deficit (AMD) stability criterion for compact systems.
- Conservative eccentricity cap of 0.3 where data is absent.

## Belt Inference Pipeline

### Inner Belt (Asteroid Analog)
- Place between innermost rocky planet and first giant (if present).
- Width scales with gap size; mass proxy ∝ stellar mass × 10⁻⁴.
- Skip if no clear gap or system architecture doesn't support it.

### Outer Belt (Kuiper Analog)
- Place beyond outermost giant planet.
- Inner edge: 1.5 × outermost giant SMA.
- Width: ~30% of inner edge distance.
- Mass proxy ∝ stellar mass × 10⁻³.
- Composition: icy volatiles, cometary nuclei.

### Belt Outputs
- `belt_type`: inner_rocky | outer_icy
- `inner_edge_au`, `outer_edge_au`, `estimated_mass_earth_masses`
- `composition_estimate`: rocky | mixed | icy
- `resource_potential`: low | medium | high (for game economy)

## Simulation-Facing Outputs

Each inferred (and observed) object is enriched with fields consumed by the simulation engine:

| Field | Computation | Used By |
|-------|-------------|---------|
| `resource_potential` | f(mass, type, belt proximity, metallicity) | Economy: extraction rates |

| `terraforming_difficulty` | f(atmosphere gap, mass deviation from Earth, temperature delta) | Colony: investment cost |

## Model Versioning & Reproducibility

| Metadata Field | Example Value | Purpose |
|----------------|---------------|---------|
| `inference_model_version` | `v1.2.0` | Track which model generated this entity |
| `inference_seed` | `48291037` | Deterministic reproduction |
| `inference_method` | `orbital_lane_spacing` | Which algorithm was used |
| `inference_timestamp` | `2026-02-21T14:30:00Z` | When inference was run |
| `world_build_id` | `wb_20260221_001` | Which world build contains this entity |

### Reproducibility Contract
- Same `(seed, model_version, input_catalog_hash)` → identical inferred entities.
- Previous inference outcomes archived for comparison.
- Diff reports generated when model version changes.
